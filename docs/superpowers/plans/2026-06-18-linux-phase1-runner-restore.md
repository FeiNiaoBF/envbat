# Linux Phase 1 Runner + Complete Restore Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` or subagent-driven development task-by-task. Steps use checkbox syntax for tracking.

**Goal:** Make PopOS/Linux setup, backup, restore, and verify use a tolerant staged execution model with clear `OK / SKIP / WARN / FAIL` summaries.

**Architecture:** Add a small Bash runner that records stage results and centralizes failure policy. Setup uses required stages for core machine readiness and optional stages for network/tool installs. Backup/restore keep separate entrypoints but adopt the same result model and restore safety rules.

**Tech Stack:** Bash, Pop!_OS/Ubuntu apt, existing `popos/*.sh` modules, JSON manifest generated by shell-safe heredoc.

---

## Implementation Scope

- Linux/PopOS only.
- Windows backup/restore remains Phase 2.
- `zsh` is required.
- oh-my-zsh/P10k, language runtimes, Docker, input method, Neovim, Chrome, and security hardening are optional.

## Acceptance Criteria

- `./popos/setup.sh` reaches a final stage summary unless a required stage fails.
- `./popos/setup.sh --repair` loads an existing profile and skips install questions.
- Optional failures are recorded as `WARN` and do not abort setup.
- `backup.sh` writes `MANIFEST.txt` and `manifest.json`.
- `restore.sh` restores user state by default, asks separately for `.ssh`, skips full package/system restore by default, and clones only missing repos.
- `verify.sh` has no `((count++))` / `((var++))` `set -e` hazards.
